0.30.0 — subtitles

Best-in-class subtitles. Switch audio and subtitle tracks instantly, load an external file in any encoding, nudge the sync and move the line where you want it, run two subtitle tracks at once, and — if you opt in with your own key — fetch subtitles online. Completely free, every feature included. No ads, no telemetry, no account — ever.

New in 0.30.0

  • Audio & subtitle track switching. A menu for each, listing every track in the file — by its name, or its language spelled out — with instant switching.
  • Load any subtitle file. SRT, ASS/SSA, WebVTT, MicroDVD, and image-based PGS/VobSub. A legacy-encoded file (Windows-1251, Shift-JIS, …) is auto-detected and converted so it reads correctly instead of as garbled text — and it tells you what it did.
  • Sync, position and size — remembered per file. Fix an out-of-sync subtitle, move it up off a burned-in caption, or make it bigger; the adjustment is there next time you open that file.
  • Two subtitles at once. Show a second track alongside the first — handy for learning a language.
  • A readability override. Force your own font, size and colour over a file's styling, for accessibility. Off by default, and honest that it overrides the author's design.
  • Opt-in OpenSubtitles fetch. Off unless you turn it on and add your own free API key. Only the title and languages you search are ever sent, over an encrypted connection; your password is never stored.

0.20.0 — core playback UI

The modern player, on top of the native surface. A glassy control bar sits under the picture and gets out of the way while it plays; the whole transport is here — seek, speed, A–B repeat, frame-step, snapshots, keyboard shortcuts, the OS media keys — and a file reopens right where you left it. Freally Player is completely free — every feature included. No ads, no telemetry, no account — ever.

New in 0.20.0

  • A glassy, auto-hiding control bar. Play/pause, frame-step, ±10s, a live clock, volume with mute, a playback-speed menu (0.25×–4.0×), A–B repeat, a chapters menu, snapshot, and fullscreen. It fades away while the picture plays and comes back the moment you move the mouse.
  • A real scrubber. Click or drag to seek anywhere, with a buffered-range bar, chapter tick marks, the A–B loop shaded in, and a timecode that follows your cursor.
  • Resume where you left off. A file reopens at the second you stopped, with the audio and subtitle tracks that were playing. A file that changed on disk starts fresh instead of dropping you in the wrong place, and the idle screen keeps a Continue-Watching row.
  • Keyboard-first, and the hardware media keys work. Space, arrows, J/K/L, volume, mute, fullscreen, frame-step and speed from the keyboard; your OS's now-playing panel shows the title and play state, and its media keys drive the player.
  • Snapshots. Save the current frame — with the subtitle overlay — to a PNG.
v0.10.1 Jul 22, 2026

0.10.1 — the first build that plays

The installers now carry the playback engine, so a downloaded build plays media out of the box — no extra libraries to install. Freally Player is completely free — every feature included: the full player, the media library, casting, and conversion. No ads, no telemetry, no account — ever.

New in 0.10.1

  • The player actually plays. The media engine ships inside the installers on Windows, macOS and Linux. Nothing to install alongside it.
  • Every language renders. The Noto typeface is bundled for Latin, Greek, Cyrillic, Vietnamese, Arabic, Hindi, Chinese (Simplified and Traditional), Japanese and Korean — including filenames and subtitle tracks, whatever script the media itself uses. Previously these could appear as empty boxes on Linux.
  • macOS ships for both chips. Separate Apple Silicon and Intel builds, each with the engine bundled.

Also in the 0.10 foundation

  • The Tauri v2 shell (Rust core + a modern React/TypeScript UI in the Havoc dark theme, with a light theme too), and the Windows build as a GUI app that opens no console window.
  • The key architecture, working: decoded video is rendered into a native, hardware-accelerated GPU surface driven through the libmpv render API and composited with the web UI, so video never travels through the interface layer. Video output runs on Windows today; the macOS and Linux surfaces are still being built, and the app says so plainly rather than showing a blank screen.
  • Playback controls wired end to end — open, play, pause, and seek — with the interface following the player's real state.
  • A first-run licence agreement, and opt-in anonymous bug reporting that never sends anything without an explicit click.
  • The cross-platform CI matrix — including a job that verifies the media engine on Windows, macOS and Linux — and a tag-triggered release workflow that produces the installers per OS.
  • This Havoc-branded site, with the roadmap to 1.0.0.
